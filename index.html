The Change Preventer

Bad code smells in this category make future changes difficult and risky.

Divergent Change

Divergent change occurs when a single class is frequently modified for different reasons.


// Smell Code
class Customer {
    
        // Some methods related to billing
        // Some methods related to shipping
    
}

Solution: Separate the concerns by creating separate classes for billing and shipping.

Shotgun Surgery

Shotgun Surgery occurs when a single change to the codebase requires edits to multiple classes or modules.


// Smell Code
class Order {
    void calculateTotal() {
        // logic to calculate order total
    }
}

class Invoice {
    void calculateTotal() {
        // logic to calculate invoice total
    }
}

Solution: Refactor related functionality into a single module or class to reduce dependencies.

Parallel Inheritance Hierarchy

Parallel Inheritance Hierarchy occurs when a new subclass is created to mirror another class hierarchy.


// Smell Code
class Shape {
    // Common properties and methods for all shapes
}

class Rectangle extends Shape {
    // Rectangle-specific properties and methods
}

class ColoredRectangle extends Color {
    // Color-specific properties and methods
}

Solution: Use composition instead of inheritance to share common behavior and reduce coupling.

The Dispensable

Bad code smells in this category represent unnecessary complexity or redundancy.

Comment

Excessive or misleading comments that don't add value to the code.


// Smell Code
// Increment i by 1
i++;

Solution: Write expressive code and remove redundant comments by using meaningful variable and function names.

Duplicate Code

Duplicate code occurs when the same code appears in more than one place in a codebase.


// Smell Code
function calculateTotal(price, quantity) {
    return price * quantity;
}

function calculateTax(price, quantity) {
    return price * quantity * 0.1;
}

Solution: Refactor duplicate code into reusable functions or modules to improve maintainability and reduce the risk of inconsistencies.

Lazy Class

Lazy class is a class that doesn't do enough to justify its existence.


// Smell Code
class Employee {
    // Minimal functionality or few methods
}

Solution: Consider merging the functionality of lazy classes into other classes or removing them altogether.

Data Class

Data class is a class that only contains data and no additional functionality.


// Smell Code
class Person {
    String name;
    int age;
}

Solution: Refactor data classes to encapsulate behavior or merge them with related classes.

Dead Code

Dead code is code that is no longer used and serves no purpose.


// Smell Code
function unusedFunction() {
    // Function that is never called
}

Solution: Remove dead code to improve code clarity and maintainability.

Speculative Generality

Speculative generality is adding unnecessary complexity to handle potential future requirements that may never arise.


// Smell Code
class Calculator {
    // Methods for complex calculations that are not currently needed
}

Solution: Refactor or remove overly generalized code to simplify the design and avoid unnecessary complexity.
